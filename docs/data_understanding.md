## 数据理解与探索总结

### 1. 数据总览
- **LC.csv**（标的特征）：328,553 条记录，21 个字段，与 `ListingId` 一一对应，覆盖借款金额、期限、利率、评级、认证、历史行为等成交时可获得信息。借款成功日期范围为 2015-01-01 至 2017-01-30。
- **LP.csv**（还款计划/记录）：3,203,276 条记录，10 个字段，记录每支标每期还款情况，与 LC 在 `ListingId` 上一一对应。recorddate 固定为 2017-02-22。
- **LCIS.csv**（投资快照）：292,539 条记录，37 个字段，包含投资额、当前状态等信息。`ListingId` 唯一数为 67,388（某投资人的投资子集），recorddate 覆盖 2016-09-30 至 2017-02-28 的月末快照。
- **关联情况**：LC 中全部 `ListingId` 均可在 LP 中找到对应期次；但 LC 与 LCIS 的交集较小（约 5.3%），因此 LCIS 主要用于补充“标当前状态”等快照字段。

### 2. 字段类型与初步质量
- LC 表字段类型较干净，主要为整型/浮点型/类别型，抽样未见缺失。
- LP 表日期列需显式转换为日期格式；`还款日期` 有约 48.7% 缺失，表示截至记录日仍未还款。
- LCIS 表中 `上次还款日期`、`下次计划还款日期` 及对应金额存在缺失（样本中约 7% 与 37%），符合快照未发生还款的情形；`标当前状态` 列包含“正常还款中”、“已还清”、“逾期中”等，也存在噪声值（如 `0`、`3.56`）。

### 3. 标签构建初探
- LP 期次级数据经过日期推断后，约 28.8% 的标至少出现一次逾期（规则：已还款但还款日期晚于到期日，或在记录日仍未还款且已过到期日）。
- LCIS 的 `标当前状态` 可直接识别“逾期中”标的，但覆盖面有限。
- **后续标签策略**：
  1. 合并 LCIS 中 `标当前状态` 识别的逾期标记；
  2. 对其余标按 LP 期次规则判定；
  3. 若存在冲突（如 LCIS 显示已还清但 LP 表示逾期），以 LP 期次判定为准，并在报告中说明该假设。

### 4. 数据清洗与一致性检查要点
- 异常年龄、历史借款指标需与业务规则比对（如年龄合理区间 18-70）。
- 认证字段为“成功/未成功”等字符串，需要标准化为二值。
- 日期字段需统一解析，提取年、月、是否月初/周末等特征。
- `借款成功日期` 与 `到期日期` 间应满足期限关系，可检测极端值。
- 金额字段（借款金额、历史成功借款金额、总待还本金等）需排查负值/极端值，并考虑 Winsorize 或对数缩放。

### 5. 特征设计初步规划
- **基础特征**：借款金额、期限、利率、评级、借款类型、首标标识、年龄、性别、各类认证标识、历史借贷行为统计。
- **衍生特征**：
  - 历史逾期率 = 历史逾期还款期数 / (历史逾期还款期数 + 历史正常还款期数)。
  - 历史还款能力指数 = 历史正常还款期数 / (历史正常还款期数 + 历史逾期还款期数)。
  - 借款杠杆比 = 借款金额 / (历史成功借款金额 + 1) 或基于评级/认证映射的收入 proxy。
  - 循环负债率 = 总待还本金 / (历史成功借款金额 + 1)。
  - 时间特征：借款成功日期所在季度、是否月初、节假日等（若可获取）。
- **交互特征**：评级与借款类型组合、认证数量统计等。

### 6. 未来分析方向
- 深入分析不同评级、借款类型在逾期上的差异，辅助模型解释。
- 评估类别不平衡情况（预期逾期率约 28%），决定是否采用采样或阈值调节策略。
- 对 LCIS 快照进行时间序列分析，观察“逾期中”转移轨迹，为模型提供时间敏感特征。
