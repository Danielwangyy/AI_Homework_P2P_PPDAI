## 项目总结报告

### 1. 项目概述
- 项目目标：基于借贷平台 LC、LP、LCIS 数据构建违约预测模型，辅助风控识别高风险借款人。
- 团队配置：客户、项目经理、数据科学家、机器学习工程师、测试工程师五个角色分工协作，并分别提交岗位报告。
- 最新成果：完成数据清洗与特征构建流水线，扩展并统一训练逻辑回归、LightGBM、XGBoost、CatBoost 四类模型，结合成本敏感阈值调优、可解释性分析与完整文档交付。

### 2. 关键成果回顾
- **数据处理**：
  - 标签融合策略结合 LCIS 状态与 LP 期次规则，生成可靠的违约标签（逾期率约 28.8%）。
  - 特征工程覆盖历史逾期行为、借款杠杆、时间衍生变量；并通过统一的清洗流水线保证数据集分层拆分与可复现性。
- **模型表现（2025-11-08 20:40 版本）**：
  - 四类模型均在成本敏感阈值（FN:FP=5:1）下评估，Recall 均超过 0.88，Precision 维持 0.37-0.42；LightGBM 验证集 F1=0.544、ROC-AUC=0.759。
  - 与历史版本相比，树模型的 Recall 略有提升，而代价是 Accuracy 与 Precision 下降；需要结合业务策略决定是否继续采用该成本权重。
  - 特征重要性与 SHAP 结果强调历史逾期率、额度利用率、借款金额是关键影响因子。
- **交付物**：
  - 代码：主入口迁移至 `src/ai_homework/` 包，包含 `pipelines/{prepare_data,train_models}.py` 与 `cli/run_pipeline.py`。
  - 数据：`data/processed/`、`data/interim/`。
  - 模型与评估：`outputs/models/`、`outputs/reports/tables/`、`outputs/reports/figures/`、`outputs/artifacts/`。
  - 文档：需求、设计、角色报告、结果总结等位于 `docs/`。

### 3. 经验与成效
- 标准化目录和配置驱动的流水线大幅提高多角色协作与复现效率。
- 训练阶段新增 Mac 平台下对 `libomp` 与中文字体的兼容处理，保障树模型与可视化稳定运行。
- 借助自动化阈值调优与统一的图表/表格输出，使模型成本假设与业务沟通更通畅。

### 4. 改进与扩展
- 数据层面：引入外部宏观经济指标、设备行为数据，丰富特征空间。
- 模型层面：继续探索集成学习或深度模型，并对成本敏感权重进行 A/B 测试。
- 解释层面：在全量 SHAP 基础上补充局部解释或稳定性分析，提升对关键特征的信心。
- 部署层面：构建批量评分/在线 API，结合 CI/CD 与监控体系，支撑生产化。
